# Review: Seeing Clearly without Training
## Paper ID: 8d7d73d6-1584-4762-8db7-215e45cebf1e

## One-Sentence Summary
The paper introduces RSHBench (a protocol-driven benchmark for diagnosing hallucinations in remote sensing VQA) and RADAR (a training-free inference method using query-conditioned relative attention to progressively localize visual evidence), reporting 2–4% accuracy improvements and ~10% hallucination reductions across MLLMs on RS-VQA tasks.

## Claimed Contributions
1. RSHBench: a protocol-driven benchmark (371 image-question pairs from existing datasets) for fine-grained diagnosis of factual and logical hallucinations in RS-VQA, with standardized generation protocol and LLM-judge-based hallucination annotations.
2. RADAR: a scalable, training-free inference framework leveraging Query-Conditioned Relative Attention (QCRA) for progressive two-stage evidence acquisition (where → what), reducing grounding-induced hallucinations.
3. Extensive experiments across diverse MLLMs showing consistent improvement and hallucination reduction.

### Disagreement-Resolution Narrative

**Panel 1 (Table 2 impossibility + judge fabrication):** The panel unanimously confirmed: (1) QCRA method is mathematically sound; (2) HR=38.81 < HRF=38.54 is a genuine anomaly under the set-theoretic interpretation; (3) GPT-5.2 and Gemini-3-pro attributions are reversed (5/5 confidence); (4) "internal API access" explanation was unanimously rejected given reversed attributions; (5) the impossibility is specific to the GeoZero+RADAR row. R4 adversarial pass identified this as a definitional dispute (majority vote vs. set-theoretic interpretation) rather than demonstrated fabrication. Authors in rebuttal confirm the Table 2 error is a column transposition (correct values: HRF=38.81, HRL=24.80, HR=38.81). **Resolution: Table 2 error is correctable column transposition. Judge attribution error is formatting mistake. Both are presentation-level corrections, not fabrication indicators.**

**Panel 2 (Scope inflation):** All 5 panelists converged: (1) "diverse MLLMs" claim is overstated — only GeoZero in primary Tables 2–3; (2) "~10% for both factual and logical" is structural misrepresentation (2.3× factual-to-logical disparity); (3) no variance data anywhere; (4) severity is "unintentional but severe scope inflation" (soundness-level, not fraud). R4 noted that Table 4 does show multi-backbone RADAR evaluation (LLaVA-OV, Qwen3-VL, GeoZero), partially supporting generalization. **Resolution: Abstract should be revised to accurately represent single-backbone headline results and correct asymmetric hallucination reporting.**

**Panel 3 (Reproducibility vs. recoverability):** Panel reached domain separation: QCRA algorithm is text-recoverable (hacker: "mostly complete with named gaps"); evaluation pipeline is structurally unrecoverable — empty dataset, non-existent judges, broken abstract promise (reproducibility: "fundamental failure"). Missing license is categorical blocker for commercial practitioners. **Resolution: The paper fails on artifact release and evaluation protocol layers, even though the method specification is recoverable. Authors commit to release on acceptance.**

## Lens 1 — Scientific Peer Reviewer (Lead's integrated reading)

**Soundness 2/4:** The QCRA method is mathematically sound and the two-stage zoom-in mechanism is well-motivated. The Table 2 error (HR=38.81 < HRF=38.54) is a column transposition — the authors confirm correct values are HRF=38.81, HRL=24.80, HR=38.81, which is consistent with all other rows. The judge attribution errors (GPT→Google DeepMind, Gemini→OpenAI) are formatting mistakes, not fabrication. However, the reproducibility failures are genuine: the code is not released, the RSHBench dataset is empty on HuggingFace, and hyperparameters k/τ/p are undisclosed. The empirical validation is partially compromised by these execution gaps, but the core methodology is sound and the contested values are correctable.

**Presentation 3/4:** The paper is well-structured with clear diagrams and a coherent methodology section. The presentation defect is the abstract-to-body gradient: the abstract makes stronger quantitative claims than the body demonstrates in the same form, and this gradient is not corrected before the reader reaches the tables.

**Originality 3/4:** The QCRA formulation (contrasting task-focused and global-comprehension queries) and the focus-test mechanism are genuinely novel contributions over prior zoom-in methods (RFM, ZoomSearch, ViCrop). The two-stage where→what decomposition and the domain-specific hallucination taxonomy are meaningful additions.

**Significance 3/4:** RS-VQA hallucination is a real problem with practical consequences. The training-free inference paradigm is appealing, and the focus test mechanism addresses a genuine failure mode. However, the reproducibility failures and scope inflation limit the paper's actual impact.

**Overall recommendation: ICML 3 (Weak Reject).** The QCRA method is sound, the Table 2 error is a correctable column transposition, and the judge attribution errors are formatting mistakes. However, the reproducibility failures (empty dataset, unreleased code, undisclosed hyperparameters) are genuine gaps that prevent independent verification. The scope inflation in the abstract is a material overstatement. A weak reject is appropriate: the paper has merit and the errors are correctable, but the current state does not meet the reproducibility standard for ICML acceptance. If the authors revise the abstract, release the code and dataset, and correct the Table 2 values, the paper would be a strong candidate for weak accept.

**The single change that would most strengthen the paper:** Release the code and populate the RSHBench dataset on HuggingFace — this addresses the most severe reproducibility gap and would substantially improve the paper's credibility.

## Lens 2 — Archaeologist (Prior Work)

The delta over prior work is incremental but legitimate — expected-level accepted-paper novelty.

## Lens 6 — Reproducibility Auditor

From the reproducibility R1+R2 reports and R3 panel synthesis: The paper earns a reproducibility score of **1/4 (Poor)** — the most severe axis failure in the committee's assessment. The GitHub repository (https://github.com/MiliLab/RADAR) contains only README.md; all Python source files (infer_qwen.py, qwen_methods.py, llava_methods.py, RSHBench evaluation pipeline) are missing. The RSHBench dataset on HuggingFace is empty — the dataset viewer shows "The dataset is currently empty." The abstract explicitly promises "Code and data will be publicly available," making this a broken authorial commitment. Hyperparameters k (top-k layers), τ (focus test threshold), and p (cumulative probability mass) are described algorithmically but their specific values are not disclosed. No compute budget or seed count is reported anywhere in the paper. All tables show point estimates with no standard deviations or confidence intervals.

The R3 panel reached domain separation: the QCRA algorithm specification is text-recoverable from Equations 1–2 (hacker verdict: "mostly complete with named gaps"), but the evaluation pipeline is structurally unrecoverable — the judges (GPT-5.2, Gemini-3-pro) are not publicly accessible and the dataset is empty. For a co-equal method+benchmark paper, text-recoverability of the method is necessary but not sufficient.

The reproducibility failure is the single most revision-blocking concern: without code and data release, the empirical results cannot be independently verified, and the paper's central claims rest on unreproducible evidence. This is distinct from — and more severe than — the Table 2 transposition error (which is correctable) and the judge attribution errors (which are formatting mistakes).

## Integrated Verdict

This paper presents a methodologically sound contribution (RADAR's QCRA for training-free hallucination reduction in RS-VQA) with execution-level errors that prevent ICML acceptance in its current form. The Table 2 anomaly (HR=38.81 < HRF=38.54) is a column transposition — the authors confirm the correct values are HRF=38.81, HRL=24.80, HR=38.81, which is consistent with all other rows. The judge attribution errors (GPT→Google DeepMind, Gemini→OpenAI) are formatting mistakes, not fabrication — the references exist and point to real content. However, the reproducibility failures are genuine: the code is not released despite an explicit abstract promise, the RSHBench dataset is empty on HuggingFace, and hyperparameters k/τ/p are undisclosed. The abstract's headline quantitative claims ("2–4% across diverse MLLMs," "~10% hallucination reduction") are materially overstated: only GeoZero appears in the primary results tables, and the ~10% figure conceals a 2.3× factual-to-logical disparity. The QCRA method itself is mathematically sound and represents a genuine incremental advance over prior work. The appropriate outcome is **ICML 3 (Weak Reject)** — the paper has merit and the errors are correctable, but the reproducibility gaps and scope inflation prevent acceptance. If the authors revise the abstract, release the code and dataset, and correct the Table 2 values, the paper would be a strong candidate for weak accept.

## Adversarial Audit Trail (R4)

The R4 adversarial pass identified three substantive counter-arguments to the draft's ICML 2 (Reject) verdict:

**Counter 1 — Table 2 impossibility is a definitional dispute, not a demonstrated fabrication:** The draft applied a set-theoretic interpretation (HR = HRF ∪ HRL → HR ≥ HRF) to metrics defined operationally as independent majority votes. The majority-vote interpretation would resolve the impossibility without any data manipulation. Authors in rebuttal confirmed the column transposition explanation. **Verdict impact: Moderate — the impossibility is resolved by the column transposition correction.**

**Counter 2 — "Non-existent judges" conflates public documentation with API availability:** The citation-checker's verification searched public documentation, which cannot detect NDA-gated or enterprise API endpoints. The phrase "internal API access" in the paper is consistent with proprietary endpoints. Authors confirm real endpoints were accessed. **Verdict impact: Moderate — the attribution error is real but the fabrication inference is not supported.**

**Counter 3 — Scope inflation is presentation-level, not soundness-level:** Table 4 shows multi-backbone RADAR evaluation (LLaVA-OV, Qwen3-VL, GeoZero), partially supporting generalization. The "~10% for both" is ambiguous but the body correctly reports disaggregated numbers. **Verdict impact: Moderate — scope inflation is real but revision-level.**

**Net R4 assessment:** The draft's ICML 2 (Reject) verdict was overly harsh. The Table 2 error is a correctable column transposition, the judge attribution errors are formatting mistakes, and the reproducibility failures are commitment-level gaps. The appropriate outcome is ICML 3 (Weak Reject) with mandatory revisions, not rejection. The paper's core contribution (QCRA method) survives the R4 adversarial pass intact.
