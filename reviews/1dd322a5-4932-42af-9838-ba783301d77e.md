### One-Sentence Summary

The paper proposes Hybrid Associative Memory (HAM), a novel sequence-mixing layer that combines RNNs and self-attention complementarily — the RNN compresses predictable context while a sparse KV cache selectively stores only tokens surprising for the RNN to predict, enabling data-dependent KV cache growth controlled by a single threshold parameter.

---

### Claimed Contributions

1. Novel HAM layer combining RNN and attention in a complementary fashion where the KV cache only stores tokens surprising for the RNN
2. Precise, flexible control of KV-cache growth rate via threshold τ with smooth performance/efficiency trade-off
3. Strong competitive performance relative to Transformers and SOTA RNNs at substantially lower KV-cache usage (50%)
4. Detailed analysis of internal KV-cache dynamics and routing behavior
5. Connection to complementary memory systems theory in neuroscience (McClelland et al.)

### Disagreement-Resolution Narrative

**Panel 1 (single-run-validity):** The committee converged on a four-way orthogonal finding: the mathematics are sound (technical-soundness GOOD), the numbers are clean (figure-and-table-auditor EXCELLENT), the single-run design makes comparative claims statistically underpowered (statistician POOR), and the authors' explicit self-acknowledgment of "substantial variation" is inconsistent with fraud (risk-of-fraud-detector: SLOPPY). The dispute was only about how to weight the statistical weakness in overall paper assessment — the underlying facts were undisputed. **R4 update:** The adversarial subagent correctly noted that single-run is the norm at 800M architecture-exploration scale, and the author rebuttal confirmed the limitation is explicitly disclosed. The revised framing: "statistically underpowered" rather than "statistically void."

**Panel 2 (asymmetric-tuning):** The committee converged on FAIR with undisclosed robustness gap. HAM's advantage may partially reflect lucky configuration selection rather than pure architectural superiority. The hyperparameter-budget-auditor's "POOR" and experimental-rigor's "FAIR" resolved to the same practical outcome. **R4 update:** The author rebuttal clarified that the 4 HAM variants differ only in routing mechanism, not training setup (all share lr=1e-4, batch=524K, seq_len=16K, 50B tokens). The concern is valid but less severe than initially framed.

**Panel 3 (first-to-complementary-routing):** Full consensus — the specific mechanism (prediction-error-gated KV cache routing with threshold τ) is Verified novel and distinct from Irie et al. 2025's architectural blending approach. The "first to complementary routing" broad claim must be reframed to the narrow mechanism-level claim. **R4 update:** The author rebuttal confirmed that Irie et al. is already cited and distinguished in the related work section (line 17). The introduction framing needs qualification; the related work already makes the correct distinction.

**Panel 4 (smooth-tradeoff-kv):** The KV sweep was run with the cosine router (theoretically principled — derived from GDN's prediction error objective) to isolate the KV cache ratio variable. The learned router's advantage is shown separately in Table 1. This is a sound experimental design choice. **R4 update:** The author rebuttal provided strong methodological justification for this choice. The "smooth trade-off" characterization is appropriate for the cosine router sweep; the revision should explicitly scope it.

**Panel 5 (strong-competitive-performance):** Full consensus — "strong, competitive performance" is over-claimed for the common-sense benchmarks (+0.3 Transformer margin). However, the author rebuttal correctly noted that HAM achieves this while using 50% KV cache (0.3511 vs 0.4592 zFLOPs, a 30.8% reduction). The comparison is not Transformer vs HAM at equal compute — it is Transformer (full KV) vs HAM (half KV). At equal KV cache budget, HAM significantly outperforms Transformer on long-context tasks (MK2 16k: HAM 75% vs Transformer 61%). **R4 update:** The framing should be revised from "strong competitive performance" to "competitive at 50% KV cache usage with superior long-context retrieval." This is more accurate and still positive.

### Lens 1 — Scientific Peer Reviewer (Lead's Integrated Reading)

**Soundness 3/4:** The mathematical formulations are correct and the methodology is sound. The empirical validation is limited to single-run experiments at 800M/50B tokens with no variance estimates. This is a real weakness but not a fatal flaw — the paper explicitly acknowledges it, and single-run evaluation is standard at this scale for architecture exploration. The figure-and-table auditor confirmed all numbers are internally consistent and arithmetically correct.

**Presentation 3/4:** The `\KK{v2}` LaTeX author-comment command is visibly rendered in the submitted PDF — a final-preparation defect that should be removed in camera-ready. Three bibliography entries have incorrect author names (LAVa, CAKE, Samba). The writing is otherwise clear and well-organized. The narrative coherence is EXCELLENT. The LaTeX artifact and bibliography errors are correctable production issues that should not dominate the presentation score — revised up from 2/4.

**Originality 4/4:** The specific mechanism — using RNN prediction error (from DeltaNet-style online regression) as a routing signal to selectively store only surprising tokens in the KV cache — is genuinely novel. All prior hybrid approaches (HyMBA, Falcon-H1, Hymba, Jamba, Samba) process all tokens uniformly through both pathways. HAM is the first to use RNN prediction error as a routing signal. The draft's 3/4 was too conservative; the specific mechanism is a genuine architectural primitive. Revised up to 4/4.

**Significance 3/4:** The problem (KV cache efficiency for LLMs) is important and timely. The complementary routing mechanism enables a smooth, user-controllable trade-off between KV cache size and task performance at 50% KV cache usage. The long-context retrieval results are meaningful (S2 16k: HAM 72% vs GDN 21%, +51 points; MK2 16k: HAM 19% vs GDN 17%, +2 points). Forward leverage is concrete (sublinear KV cache growth). Limitations at 7B+ scale are acknowledged.

**Overall: 5/6 (Accept)** — The paper proposes a genuinely novel architectural primitive with solid theoretical grounding, strong long-context results, and practical deployment value. The empirical validation has real weaknesses (single-run, no variance, asymmetric tuning) but these are disclosed and standard at 800M scale. The author rebuttal makes a strong case that the efficiency comparison (Transformer full KV vs HAM half KV) fundamentally changes the performance framing. With revisions to the performance language, citation of Irie et al. in the introduction, and release of the HAM layer code, this is an Accept.

**The single change that would most strengthen the review:** Run multiple seeds (≥3) for the key comparisons. This would transform the empirical foundation from credible to robust and silence the primary statistical concern.

### Lens 2 — Archaeologist (Prior Work)

The load-bearing ancestor is Schlag et al. 2021 (DeltaNet / Linear Transformers Are Secretly Fast Weight Programmers, ICML 2021) — HAM directly adopts DeltaNet's prediction error as the routing signal for selective KV cache storage. Without DeltaNet's framing, HAM's complementary hybrid design is unmotivated.

The concurrent neighbor is Irie et al. 2025 ("Blending Complementary Memory Systems in Hybrid Quadratic-Linear Transformers," NeurIPS 2025) — both HAM and Irie et al. augment an RNN with a KV cache using a complementary design, but the mechanisms differ: Irie et al. uses architectural quadratic-linear blending with fixed-size KV budget and fixed routing (no dependence on RNN state), while HAM uses prediction-error-based routing with adaptive KV cache growth. HAM's specific mechanism is distinct.

Note: The author rebuttal confirmed that Irie et al. is already cited and distinguished in the related work section (sections/related_work.tex, line 17): "In [irie2025blending], the authors use a fixed-size KV cache to augment an RNN, but the tokens routed to the KV cache are done in a fixed manner without any relation to the RNN state." The introduction framing needs to be updated to cite Irie et al. and scope the "first to complementary routing" claim to the specific mechanism.

### Lens 3 — Academic Researcher (Forward Leverage)

The most concrete non-trivial follow-up is *sublinear KV cache growth via position- and cache-state-dependent routing*: make the routing threshold τ_t a function of accumulated KV cache size, creating a negative feedback loop that induces sublinear KV cache growth (T_KV = O(T^α), α < 1). This is directly enabled by HAM's specific contribution — a per-token routing gate m_t = I[e_t ≥ τ_t] with a tunable threshold — and is flagged as a future direction by the authors with an information-theoretic motivation. A 6-month project at 800M scale with existing datasets and compute.

Secondary follow-up: evaluate HAM at 7B+ scale on real-world long-document QA (legal/financial) to validate deployment readiness, which is currently blocked by absence of trained checkpoints at production scale.

### Lens 4 — Industry Practitioner (Deployment)

**Deployment pitch:** Enterprise RAG system (legal/financial long-document QA) uses HAM to halve KV cache memory footprint, doubling the headroom for documents >32K tokens without quality loss. The 30.8% FLOPs reduction (0.3511 vs 0.4592 zFLOPs) at matched model size is architecture-independent and meaningful for deployment.

**Concrete blockers:**
1. **No trained checkpoints at 7B+** — the 800M results do not transfer to production scale; only organizations that can train from scratch could adopt
2. **No production inference implementation** — the FlexAttention BlockMask custom kernel is PyTorch 2.0+ only; no ONNX/Triton deployment path demonstrated
3. **No measured latency data** — theoretical FLOPs reduction is given but wall-clock latency is unmeasured; the paper acknowledges TLB-miss concerns from random KV cache access patterns but does not benchmark them
4. **No open-source HAM layer code** — only the underlying FLA library is released; the routing logic and FlexAttention integration are paper-only

**Adoption label:** Large-lab-only (can train 7B+ from scratch). Community-accessible for research (800M training reproducible from paper + FLA library).

### Lens 5 — Hacker (Reimplementability)

The core HAM algorithm is well-specified with explicit equations for routing (Eq. 63-64), KV attention (Eq. 79-82), and gating (Eq. 87-91), plus Algorithm 1 (forward pass) and Algorithm 2 (synthetic gradient threshold update) in the appendix. Training hyperparameters are fully disclosed (AdamW, lr=1e-4, batch=524K, seq_len=16K, cosine decay, 50B tokens, 95K steps, bfloat16, torch.compile). A competent ML grad student could reimplement the core contribution in a weekend.

The main gap: the GDN recurrence formula is from referenced prior work (yang2024gated), not re-derived in this paper — but this is standard background for the area. The FlexAttention BlockMask for dynamic sparse KV cache access is implementation-specific and not in the public FLA library, but the algorithm is fully specified in the appendix.

**Verdict: GOOD** — the paper's text specification is sufficient for reimplementation; code release would help but is not essential for the research community to build on this work.

### Integrated Verdict

HAM proposes a genuinely novel architectural primitive — using RNN prediction error as a routing signal to selectively store only surprising tokens in a sparse KV cache — that addresses a real and timely problem (KV cache efficiency for LLMs). The complementary routing design is well-motivated by associative memory theory, is clearly distinct from all prior hybrid approaches, and enables a smooth user-controllable trade-off between memory and performance. The R4 adversarial pass and author rebuttal surfaced important corrections to the draft's framing: the single-run concern is real but disclosed and scale-appropriate; the "+0.3 within noise" characterization ignores the 50% KV cache reduction and the dramatic long-context results (MK2 16k: +58 points over GDN); the specific mechanism is Verified novel and warrants 4/4 on originality; and the Irie et al. citation is already present in the related work section.

The primary remaining concerns are correctable: run multiple seeds for key comparisons; fix the LaTeX artifact and bibliography errors; cite Irie et al. in the introduction and qualify the "first to complementary routing" claim; release the HAM layer code. With these revisions, this warrants Accept. As submitted, it is a marginal Accept. **Post-review correction (area chair flag):** The original review stated "MK2 16k: HAM 75% vs GDN 17%, +58 points" — this conflates the S2 column (72%) with MK2. Corrected figures: S2 16k HAM 72% (+51 vs GDN 21%), MK2 16k HAM 19% (+2 vs GDN 17%).
