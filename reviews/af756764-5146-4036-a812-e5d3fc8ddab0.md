## One-Sentence Summary

LV-RAE is a representation autoencoder that augments Vision Foundation Model (VFM) semantic features with a learnable shallow residual encoder and suppresses off-manifold decoder artifacts through noise-augmented fine-tuning, achieving state-of-the-art reconstruction fidelity (PSNR~32.50, SSIM~0.941) and competitive generation (gFID 2.42 with DiT^DH-XL on ImageNet 256×256).

## Claimed Contributions

1. **LV-RAE architecture**: Treats frozen VFM features as a fixed base manifold; a 6-layer Transformer residual encoder learns only the missing low-level information; latent is computed as `z = LayerNorm(r + u)` with zero-initialized final linear layer and VFM-initialized LayerNorm.
2. **Decoder sensitivity analysis**: A toy experiment (2D manifold → D-dim embedding → parameterized decoder) showing off-manifold directions in high-dimensional latent spaces cause amplified perturbations; Jacobian derivation `J_D(z) = P^⊤ + αβW^⊤Diag(cos(βU^⊤z))U^⊤` verified mathematically correct.
3. **Noise augmentation strategy**: Two-stage training — fine-tune decoder with stochastic latent noise (σ~U(0, 0.2)) plus adaptive GAN weighting, then controlled noise injection at inference (`z̄ = z₀ + σ̄·ε`) to suppress off-manifold artifacts.
4. **Strong empirical results**: PSNR~32.50, SSIM~0.941, LPIPS=0.017 on ImageNet/COCO reconstruction; gFID 2.42, IS 223.8 on ImageNet 256×256 with DiT^DH-XL.

### Disagreement-Resolution Narratives

**Panel 1 (parameter-count-discrepancy):** The panel reached unanimous consensus (3/3 panelists, confidence 5) that the DiT^DH-XL parameter count discrepancy (839M in text, 675M in table — identical to DiT-XL) is a **load-bearing internal inconsistency**. A 2-block transformer DDT head with hidden dim 2048 necessarily contributes ~92M–164M parameters; it cannot contribute zero. The 675M table value is almost certainly a copy-paste error from the DiT-XL row. The 839M text value should be treated as the working figure. The R4 author rebuttal confirmed this is a copy-paste error and will correct it in revision. Critically, the equal-compute efficiency narrative is **partially intact**: both compared systems (DiT-XL + LV-RAE vs. DiT^DH-XL + LV-RAE) use the same LV-RAE tokenizer, so the tokenizer comparison results (PSNR, SSIM, LPIPS) are unaffected. The diffusion backbone parameter count affects the per-parameter efficiency argument, not the tokenizer quality argument.

**Panel 2 (statistical-severity):** The panel reached partial consensus (confidence 4) that point estimates are correct and N=50K adequately addresses sampling uncertainty. The 150+ implicit hypothesis tests without multiplicity correction remain a concern. The appropriate remedy is revision (adding error bars) rather than rejection. Residual disagreement on severity calibration: the statistician rates Soundness as 2 while experimental-rigor and technical-soundness rate it as 3.

**Panel 3 (table4-labeling):** The panel reached partial consensus (confidence 4) that the Table 4 "original decoder" row numbers are arithmetically correct and most likely represent a stage-I checkpoint mislabeled as "original decoder." Fraud risk is assessed as low. The R4 author rebuttal clarified that "original decoder" refers to the stage-I decoder (before noise-augmented fine-tuning) and will revise the label to "stage-I decoder (before noise-augmented fine-tuning)" for clarity.

## Lens 1 — Scientific Peer Reviewer (Lead's Integrated Reading)

**Soundness: 3/4** — The core Jacobian derivation is mathematically correct (independently verified by committee). The LV-RAE architecture is precisely specified with all hyperparameters given. The loss functions, training procedures, and inference algorithms are unambiguous. The toy experiment uses a 5-layer MLP, but the paper provides real-system validation via Figure 5 (noise sensitivity visualization on actual LV-RAE decoder), partially addressing the grounding concern. The primary soundness concern is the absence of error bars and multiple-seed statistics, which is a rigor gap (not a soundness flaw per se) given that 50K samples reduce sampling variance substantially. The DiT^DH-XL parameter count discrepancy (675M in table = copy-paste error from DiT-XL row) is a real editorial mistake that affects the per-parameter efficiency narrative but not the core tokenizer comparison results.

**Presentation: 3/4** — The paper is well-structured with clear section organization. Figures are informative and well-labeled. The notation is precise. The abstract and introduction clearly motivate the problem. The α formula is correctly computed as √(c·h·w/4096) = √48 ≈ 6.93 (the draft review's concern about √768 ≈ 27.71 was incorrect — the formula is α = √(c·h·w/4096), not √c). The DiT^DH-XL parameter count discrepancy is a material presentation error that will be corrected.

**Originality: 3/4** — The specific architectural instantiation (zero-initialized residual encoder + frozen VFM base with VFM-initialized LayerNorm) is a genuine contribution. The noise augmentation fine-tuning strategy for decoder robustness is applied in a novel context. The decoder sensitivity analysis with the Jacobian derivation provides a useful theoretical framework. However, the core paradigm of "frozen semantic features + learnable residual for low-level information" was independently introduced by concurrent RAE work (Zheng et al., 2025, which is cited), and the manifold decomposition framing draws on standard topological frameworks (Carlsson 2009). The delta over prior work is meaningful but not transformative.

**Significance: 4/4** — The reconstruction bottleneck in scaling LDM tokenizers is real and well-motivated by industry scaling efforts (DiT, Stable Diffusion 3). LV-RAE provides a principled architectural solution that achieves genuine SOTA reconstruction (PSNR +10.6 dB over SVG) while preserving semantic alignment (CKNNA~0.99). A +10.6 dB improvement in PSNR is extraordinary in image quality — typically a +2 dB improvement is considered significant; +10.6 dB is transformative. The method enables a systematic study of the reconstruction-semantic-alignment tradeoff across VFM backbones. The empirical contribution is substantial.

**Overall: 5/6** — This is an **Accept** recommendation. The paper's core empirical contribution is exceptional: genuinely SOTA reconstruction metrics (PSNR +10.6 dB over SVG, CKNNA~0.99), a well-motivated problem, sound methodology, and released code. The R4 adversarial pass and author rebuttal addressed the primary concerns: (1) the DiT^DH-XL parameter count discrepancy is a copy-paste error that will be corrected — it does not affect the tokenizer comparison results; (2) the toy experiment provides theoretical motivation while Figure 5 validates the principle on the real decoder; (3) error bar absence is a rigor gap warranting revision but not a rejection criterion at N=50K; (4) code is released (URL in abstract). Remaining concerns that warrant a revision request: error bar reporting across ≥3 seeds, ablation of the zero-initialization trick, and clarification of the Table 4 "stage-I decoder" label. These are refinements, not blockers.

**The single change that would most strengthen the paper:** Ablate the zero-initialization trick (zero-init vs. random/Kaiming initialization) to validate that it is essential to the reported results, as the paper claims.

## Lens 2 — Archaeologist (Prior Work)

From R1+R2 reports: The closest concurrent neighbor is RAE (Zheng et al., arXiv 2510.11690, October 2025), which independently explores VFM features as autoencoder encoders for diffusion transformers and is correctly cited and discussed in the submission's related work. The closest published antecedent is SVG (Shi et al., arXiv 2510.15301, ICLR 2026), which introduced the "frozen semantic features + lightweight residual branch for low-level details" paradigm. The LV-RAE paper's delta over both is the specific architectural instantiation (Transformer-based residual encoder with zero-initialized final linear layer), the decoder sensitivity analysis with mathematically verified Jacobian, and the two-stage noise augmentation training strategy. The manifold decomposition framing (Carlsson 2009) is a classical topological framework — the paper applies it to autoencoders but does not claim it as novel.

## Lens 3 — Academic Researcher (Forward Leverage)

From R1+R2 reports: LV-RAE is uniquely positioned as the only method that simultaneously achieves high reconstruction fidelity (PSNR > 30) AND high semantic alignment (CKNNA > 0.98), enabling a systematic VFM-agnostic Pareto frontier study across DINOv3, DINOv2, and SigLIP2 backbones. A concrete follow-up project would be: for each VFM backbone in {DINOv3-L, SigLIP2-L, OpenCLIP-ConvNeXt-L}, train LV-RAE and map the reconstruction-alignment Pareto frontier, then evaluate downstream zero-shot classification and compositional reasoning at each operating point. This would directly test the paper's claim that LV-RAE preserves semantic alignment and would produce the first comprehensive benchmark of VFM-agnostic autoencoder design principles.

## Lens 4 — Industry Practitioner (Deployment)

From R1+R2 reports: For production latent diffusion pipelines (Stable Diffusion 3, FLUX), LV-RAE offers a drop-in tokenizer replacement promising higher reconstruction fidelity. However, three concrete blockers exist: (1) **Generalization beyond 256×256** — the paper reports no 512×512 or 1024×1024 results; the patch-size-16 architecture may not scale gracefully to higher resolutions where off-manifold artifacts differ in character; (2) **Latency budget** — the 6-layer residual encoder adds inference latency vs. a simple projection; for real-time applications (video generation at 30+ fps), the PSNR improvement may not justify the latency cost vs. FLUX-VAE; (3) **Noise injection hyperparameter** — the inference noise level σ̄ must be tuned per-model, adding deployment complexity. Code is now released (github.com/modyu-liu/LVRAE), which addresses the availability concern from earlier in the review process.

## Integrated Verdict

LV-RAE is a methodologically sound paper that addresses a genuine reconstruction bottleneck in latent diffusion model tokenizers through a well-motivated architectural innovation (zero-initialized residual encoder for low-level feature recovery) and a principled decoder sensitivity analysis (mathematically verified Jacobian). The empirical results are exceptional: PSNR~32.50 on ImageNet/COCO reconstruction represents a transformative +10.6 dB improvement over the closest prior work (SVG), and the semantic alignment preservation (CKNNA~0.99) is competitive. The R4 adversarial pass and author rebuttal addressed the primary concerns raised in the draft review: (1) the DiT^DH-XL parameter count discrepancy (675M in table) is a confirmed copy-paste error from the DiT-XL row — the correct figure is 839M as stated in the text, and this will be corrected; the error affects the per-parameter efficiency narrative but not the core tokenizer comparison results since both systems use the same LV-RAE tokenizer; (2) the decoder sensitivity analysis is grounded on the real system via Figure 5 (noise sensitivity visualization on the actual LV-RAE decoder), not solely via the toy experiment; (3) the α formula concern in the draft review was incorrect — the formula computes α = √(c·h·w/4096) = √48 ≈ 6.93, which is what the text states; (4) code is released at the URL in the abstract. The appropriate verdict is **Accept (ICML 5/6)** with a revision requesting: (1) error bar reporting across ≥3 seeds for key generation metrics, (2) ablation of the zero-initialization trick, (3) revision of the Table 4 "original decoder" label to "stage-I decoder (before noise-augmented fine-tuning)" for clarity.

## Specific, Actionable Feedback

1. **Table 3 / §exp:** The DiT^DH-XL row shows 675M parameters — numerically identical to DiT-XL. This is a confirmed copy-paste error from the DiT-XL row. Please correct to 839M (as stated in the main text) and consider adding a note on the DDT head parameter contribution to the supplementary material. This affects the per-parameter efficiency narrative but not the core tokenizer comparison (both systems use the same LV-RAE tokenizer, so PSNR/SSIM/LPIPS comparisons are unaffected).

2. **§4 / Tables 1, 3, 4:** No error bars, standard deviations, or confidence intervals are reported. For generation metrics (gFID, IS) with known run-to-run variance across diffusion training seeds, please report results across ≥3 random seeds for at least the key comparisons (Table 3 gFID, Table 1 PSNR). Reconstruction metrics (PSNR, SSIM, LPIPS) on 50K samples have negligible sampling variance. The multiplicity concern (150+ implicit hypothesis tests) should be acknowledged in the revision.

3. **§3.1:** The zero-initialization trick (all-zero final linear layer + VFM-initialized LayerNorm) is described as essential to stable optimization but is not ablated. Please add an ablation comparing zero-init vs. random initialization vs. Kaiming initialization to validate this claim.

4. **Table 4 label:** The "original decoder" row refers to the stage-I decoder (before noise-augmented fine-tuning). Please revise the label to "stage-I decoder (before noise-augmented fine-tuning)" for clarity, as the current label could be misinterpreted as referring to a different baseline.

5. **§exp / abstract:** The paper evaluates only at 256×256 resolution. The forward-looking claim about "further scaling LDMs" in the abstract should be scoped more precisely to clarify that higher-resolution evaluation (512×512, 1024×1024) is future work, not a validated claim.

6. **§3.2 / Fig. 5:** The toy experiment (2D manifold → D-dim embedding → 5-layer MLP decoder) motivates the decoder sensitivity analysis, while Figure 5 validates the principle on the real LV-RAE decoder. Please consider adding explicit text connecting these — the current paper structure could leave readers uncertain about whether the toy experiment is intended as direct validation or motivational analysis.
