## One-Sentence Summary

SoLA achieves reversible lifelong model editing by encapsulating each edit as an independent frozen LoRA module activated by semantic routing (kNN on stored keys), enabling precise rollback by removing a routing key — the first demonstrated reversible editing capability in the model-editing literature, though the empirical evidence for the rollback capability is thin (N=5 examples) and the statistical execution falls below ICML standards.

## Claimed Contributions

1. Semantic routing-based LoRA (SoLA): independent frozen LoRA per edit, activated via query-key distance matching
2. First reversible rollback editing capability in the literature (removing a routing key restores original behavior)
3. End-to-end decision-making via integrated master decision layer (no auxiliary routing network needed)
4. Avoids semantic drift from cluster updating (MELO's weakness) and catastrophic forgetting from parameter sharing (ELDER's weakness)
5. Strong empirical performance on SCOTUS, zsRE, and hallucination correction benchmarks

### R3 Panel Resolutions

**Panel 1 — ARR Metric Definition Inconsistency:** The paper defines ARR as a [0,1] proportion in §3 but Table 2's "ARR" column contains values (4.93–412) numerically consistent with perplexity (PPL). The panel converged on "Concerning presentation inconsistency" — the paper partially discloses PPL substitution in §3 line 321 and Table 2 caption, but the §3-to-Table-2 definitional bridge is absent. Severity is "Concerning, not Load-bearing" — main claims rest on ERR/TRR in Table 1, not ARR in Table 2. The directional ambiguity (ARR implying higher-is-better, PPL being lower-is-better) is real but mitigated by caption disclosure. **Post-R4 update:** Author rebuttal commits to adding the definitional bridge sentence in §3 ("For hallucination correction, ARR is instantiated as PPL, where lower is better"). This resolves the concern for the camera-ready.

**Panel 2 — "First Reversible Editing" Broad vs. Narrow:** Narrow form (SoLA demonstrates reversible rollback editing) = **Demonstrated** — uncontested, Table 5 provides clean empirical evidence. Broad form (no prior work can do this) = **Supported but Contested** — no prior published work demonstrates rollback capability in GRACE/MELO/ELDER, but the paper does not explicitly test GRACE's rollback potential. The architectural distinction is clear: GRACE applies edits directly to hidden states (key deletion removes retrieval but not weight delta), while SoLA's frozen LoRA makes rollback clean. **Post-R4 update:** Author rebuttal commits to explicit GRACE comparison and reframing as "first published demonstration of reversible rollback editing via frozen-LoRA + key removal." The narrow claim is solid.

**Panel 3 — Text-Table Discrepancy (+3% ERR):** The contribution list explicitly says "+3% ERR over MELO on SCOTUS" but Table 1 ERR column shows +1pp (0.97 vs 0.96). The TRR column (0.95 vs 0.92 = +3.3pp) provides a mathematically defensible basis for "3%." Panel split: contribution list = **factual misstatement** (ERR named, ERR shows +1pp only); §4.2 prose = **presentation imprecision** (TRR supports ~3%, but undisclosed). **Post-R4 update:** Author rebuttal commits to correcting the contribution list to "+1pp ERR (+3pp TRR)" — this is a straightforward one-sentence fix.

## Lens 1 — Scientific Peer Reviewer (Lead's Integrated Reading)

**Soundness: 2/4.** The core methodology (independent frozen LoRA per edit + semantic routing + rollback via key removal) is mathematically sound and well-motivated against MELO's cluster-drift failure mode. The distance function dist(·) is unspecified — the adversarial subagent correctly notes this is reconstructable (cosine distance on normalized embeddings is standard in model-editing literature), and the author rebuttal commits to specifying it in the camera-ready. The master decision mechanism is conceptually simple (Equation 3) but never ablated against an auxiliary routing baseline — this is a real experimental gap, though the architectural rationale (reusing the distance-to-nearest-key signal already computed for routing, eliminating the need for a separate network) is plausible. The absence of variance reporting, statistical significance tests, and multiple-comparison correction across 156+ implicit comparisons makes all comparative claims statistically uninterpretable. DeepSeek-R1-8B's ERR=1.07 is not a reporting error — the author rebuttal correctly identifies this as a definitional conflation: PPL-based ERR uses ratio framing (lower is better, values >1 indicate degradation on already-correct sentences), while [0,1] bounded ERR applies only to SCOTUS/zsRE tasks.

**Presentation: 2/4.** The ARR metric definition inconsistency between §3 (defines ARR as [0,1] proportion) and Table 2 (shows PPL values 4.93–412 for hallucination correction) is real but partially mitigated by §3 line 321 ("for the hallucination correction task...using standard average complexity (PPL)") and Table 2's caption ("All the scores are PPL metric"). The definitional bridge (§3 never explicitly says "ARR for hallucination correction = PPL") is missing, creating directional ambiguity for skimming readers. The contribution list's "+3% ERR" attribution is imprecise (TRR column supports ~3%, ERR shows +1pp) — author rebuttal commits to the fix "+1pp ERR (+3pp TRR)". The paper's writing is otherwise clear and well-structured.

**Originality: 3/4.** The independent frozen LoRA per edit design is genuinely novel relative to MELO (shared LoRA + cluster-updating) and ELDER (shared MoE). The semantic routing mechanism for edit activation is new. The reversible editing capability is corroborated as first-in-literature by multiple independent subagents and the R3 Panel 2. The routing mechanism is standard kNN and the master decision mechanism is a straightforward architectural choice — neither is novel at the algorithm level.

**Significance: 3/4.** The reversible editing primitive is a genuine first and enables follow-up work (SoLA-Safe: reversible safety editing) that neither MELO nor ELDER can support. The O(N) routing cost is a scalability concern for production deployment, and the absence of released code limits immediate community adoption. The paper's efficiency claim (parameter-count reduction, Fig 1) is accurate and supported — SoLA uses only 0.08M additional parameters vs. alternatives. The paper does NOT claim "1.5x inference speedup" anywhere in its body; this number appears only in the Impact Statement and is not supported by latency data. The draft review incorrectly attributed this as a paper flaw — it is a draft error. The paper's actual efficiency framing (parameter efficiency) is legitimate.

**Overall: 4/6 (Weak Accept).** The paper's primary contribution — reversible lifelong model editing via independent frozen LoRA + semantic routing — is genuine, well-motivated, and (narrowly) first-in-literature. The reversible editing capability is the paper's most significant contribution and is corroborated by multiple independent assessments. The empirical execution falls below ICML standards (zero variance, zero significance tests, 156+ uncorrected comparisons), the master decision mechanism is una blated, and the rollback demonstration uses only five examples for the headline capability claim. The presentation issues (ARR metric naming, contribution-list precision) are real but fixable in camera-ready. The significance of the contribution outweighs the severity of the execution gaps for a Weak Accept recommendation. The most impactful camera-ready improvements are: (1) statistical methodology (variance across seeds, significance tests, multiple-comparison correction); (2) master decision mechanism ablation; (3) explicit dist(·) specification; (4) contribution-list metric correction.

## Lens 2 — Archaeologist (Prior Work)

SoLA's frozen-module design directly addresses MELO's documented failure mode (cluster-updating causes semantic drift) — this is the paper's load-bearing anchor. The paper shows quantitatively (Fig 2) that cluster mismatches increase with MELO's cluster updates, motivating SoLA's frozen design choice. ELDER (AAAI 2025) is under-discussed in the related-work section, appearing only as a cited baseline without methodological differentiation. ELDER's shared MoE parameters make per-edit rollback non-selective, which is architecturally distinct from SoLA's approach — the author rebuttal commits to adding an explicit architectural distinction paragraph. The "first reversible editing" claim survives aggressive prior-art search (arXiv queries for "reversible model editing," "rollback editing," "undo knowledge editing" return no relevant results), but GRACE's rollback potential via key deletion was never explicitly tested or ruled out by the paper — this is a literature comparison gap. The narrow form (first to demonstrate reversible rollback editing) is defensible and uncontested.

## Lens 3 — Academic Researcher (Forward Leverage)

The most concrete follow-up enabled by SoLA is SoLA-Safe: reversible safety editing for LLMs, where safety interventions can be precisely rolled back without affecting related knowledge. Neither MELO (cluster drift makes per-edit revocation unreliable) nor ELDER (shared parameters make rollback non-selective) can support this use case. A second follow-up: routing-aware model editing benchmarks that evaluate edit locality and interaction effects under concurrent multi-edit scenarios — SoLA's architecture makes these measurable in ways prior methods do not. A third: investigating whether the semantic routing mechanism can be extended to non-editing tasks (e.g., dynamic module composition for multi-task models). The O(N) routing cost is a known limitation that follow-up work can address through approximate nearest-neighbor methods or hierarchical routing.

## Lens 4 — Industry Practitioner (Deployment)

SoLA is not yet production-ready. Three concrete blockers: (1) the O(N) routing cost is uncharacterized at production scale (N edit keys means N distance computations per forward pass — for a model with 1000+ edits this is prohibitive); (2) no code or weights are released, preventing internal evaluation (author rebuttal commits to code release upon acceptance); (3) the concurrent multi-edit scenario is unstudied — in production, multiple edits may be active simultaneously and their interaction under semantic routing is unknown. The master decision mechanism (Section 3.4) claims end-to-end decision-making without an auxiliary routing network, but no experiment compares against an auxiliary router baseline — this is a camera-ready improvement the author rebuttal commits to adding. The paper's efficiency claim (parameter-count reduction) is supported by Figure 1 and is accurate — SoLA uses only 0.08M additional parameters vs. alternatives.

## Lens 5 — Hacker (Reimplementability)

A competent ML grad student could reimplement the core frozen-LoRA-per-edit design from the paper text in a weekend — the LoRA formulation (Equation 1) is clear and standard. The rollback mechanism (key removal → weight delta removal) is straightforward and well-described. The main blocker is the distance function dist(·), which is unspecified in Section 3.3. The adversarial subagent correctly notes that standard practice in semantic routing / model-editing literature uses cosine distance on normalized embeddings, and the author rebuttal commits to adding this specification in the camera-ready ("dist(q, k_i) = 1 - (q · k_i) / (||q|| ||k_i||) on ℓ2-normalized vectors"). The α routing threshold is presented as a hyperparameter without justification or sensitivity analysis — the author rebuttal does not address this, but it is a minor gap. The semantic routing mechanism itself is standard kNN, so the novel contribution is the architectural composition (frozen LoRA + routing + decision layer), not the individual components.

## Integrated Verdict

SoLA's core contribution — independent frozen LoRA per edit with semantic routing activation and precise rollback via key removal — is a genuine, well-motivated innovation that addresses a real gap in the model-editing literature. The reversible editing capability is the paper's most significant contribution and is corroborated as first-in-literature by multiple independent assessments. The empirical execution falls below ICML standards in three measurable ways: zero statistical tests or variance reporting across 156+ implicit comparisons, an una blated master decision mechanism that is one of three named architectural contributions, and a rollback demonstration on only five examples for the headline capability claim. The presentation has a metric-definition inconsistency (ARR as [0,1] proportion vs. Table 2's PPL values for hallucination correction), a contribution-list precision issue ("+3% ERR" when ERR shows +1pp but TRR supports +3pp), and a missing dist(·) specification. All of these are fixable in camera-ready. The adversarial subagent raised a valid point that the paper's parameter-count reduction efficiency claim (Fig 1) is accurate and supported — the "1.5x inference speedup" number does not appear in the paper body and should not have been attributed as a paper flaw in the draft. The author rebuttal is substantively strong: it correctly identifies the DeepSeek ERR=1.07 issue as a definitional conflation (PPL-ratio framing, not a [0,1] proportion), commits to specific camera-ready fixes for every substantive concern, and provides the architectural rationale for the master decision mechanism and the GRACE distinction. The significance of the contribution outweighs the severity of the execution gaps for a Weak Accept recommendation.

## Specific, Actionable Feedback

1. **Statistical methodology (Section 4):** Report mean ± std across at least 3 seeds for all comparative claims. Apply Benjamini-Hochberg correction for the 156+ implicit comparisons. Without this, the comparative claims (SoLA vs. MELO/ELDER) are statistically uninterpretable. This is the single most important fix.

2. **Master decision mechanism ablation (Section 4.3):** Add an experiment comparing the integrated master decision layer against a baseline with an auxiliary routing network (as used in MELO/ELDER). The paper claims "no auxiliary routing network needed" but provides no ablation supporting this architectural choice. The design rationale (reusing the distance-to-nearest-key signal) is plausible but must be validated empirically.

3. **Distance function specification (Section 3.3):** Add: *"We use cosine distance: $\mathrm{dist}(\bm{q}, \bm{k}_i) = 1 - \frac{\bm{q} \cdot \bm{k}_i}{\|\bm{q}\|\|\bm{k}_i\|}$, where $\bm{q}$ and $\bm{k}_i$ are $\ell_2$-normalized before computation."* This resolves the reproducibility concern and is standard in the semantic routing literature.

4. **"First reversible editing" reframe (Section 1 / Contribution list):** Change from "first reversible model editing capability" to "first published demonstration of reversible rollback editing via frozen-LoRA + key removal." Add explicit discussion: GRACE's key deletion removes the edit from routing but does NOT reverse the underlying weight update (GRACE applies edits to hidden states directly), while SoLA's frozen LoRA makes rollback clean and selective. This closes the literature comparison gap.

5. **Contribution list correction:** Change "+3% ERR over MELO on SCOTUS" to "+1pp ERR (+3pp TRR, +2pp average) over MELO on SCOTUS" to accurately reflect which metric supports which improvement figure.

6. **ARR metric clarification (Section 3):** Add to the ARR definition: *"For the hallucination correction task, ARR is instantiated as perplexity (PPL), where lower values indicate better retention of correctly generated content."* This resolves the directional ambiguity (ARR as [0,1] implies higher-is-better; PPL is lower-is-better).

7. **Rollback demonstration scale (Table 5 / Section 4.4):** Expand from N=5 to at least 20 examples across two datasets (zsRE + SCOTUS) to demonstrate generalizability of the rollback mechanism. The current N=5 is clean and mechanistically informative but insufficient for a headline capability claim.

8. **Code release:** Release source code and model weights upon acceptance. This is the single most impactful action for community adoption and is standard for ICML empirical papers. The author rebuttal commits to this.

9. **ELDER related-work paragraph (Section 2):** Add architectural distinction: ELDER's shared MoE LoRA modules are continuously updated across edits, making per-edit rollback non-selective. SoLA's independent frozen LoRA design directly enables selective rollback that ELDER cannot support.

## Questions for Authors

1. What is the exact formula for the distance function dist(·) in Section 3.3? Cosine vs. Euclidean vs. L1 distance produces materially different routing behavior — without this specification, independent replication is not possible.

2. Can you confirm that the DeepSeek ERR=1.07 value in Table 2 uses a PPL-ratio formula (lower is better, values >1 indicate degradation on already-correct sentences) distinct from the [0,1] bounded ERR used for SCOTUS/zsRE tasks? This would resolve the apparent impossibility.

3. Why was the master decision mechanism not ablated against an auxiliary routing baseline? The paper claims "no auxiliary routing network needed" but provides no experimental support for this architectural choice.

4. Can you clarify whether the rollback evaluation (Table 5) uses the same five examples across all rollback experiments, or whether different subsets of examples are tested for each rollback condition? If the same five examples are reused, does the model have any prior familiarity with these specific examples?

5. What is the planned statistical analysis for the camera-ready? Will you report variance across seeds, apply multiple-comparison correction, and add significance tests for the comparative claims in Table 1?
