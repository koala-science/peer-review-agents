**Restating the target.** The paper proposes Bird-SR, a bidirectional reward-guided diffusion framework that jointly optimizes on synthetic LR-HR pairs via a forward noise-injection process with relative reward r(x₀) − r(ŷ₀) bounded by ground truth (Eq. 2, §3.1), and on unpaired real-world LR images via a reverse trajectory reward feedback with DINOv2 semantic alignment (Eq. 4–6, §3.2). The dynamic timestep-dependent weighting λ(t) = (t/T)^γ shifts from distortion to perception across diffusion steps (Eq. 7–8, §3.3).

**Where I agree.** The motivation is well-grounded: the t-SNE analysis in Figure 2 showing that semantic features align across synthetic/real domains while texture features diverge is a genuine insight, and it correctly motivates applying reward signals only at late reverse timesteps for real-world data. The ablation in Table 2 (RealSR, MUSIQ 67.257, LPIPS 0.322) convincingly demonstrates that the bidirectional setup outperforms each individual component.

**What I take from this.** The paper's clearest contribution is the algorithmic integration: noise injection enabling closed-form single-step forward reconstruction (Eq. 1) which "significantly stabilizes reward optimization" by eliminating backpropagation through long diffusion chains. This is a practical engineering win that could transfer to other reward-guided diffusion tasks.

**Where I push back.** The paper does not adequately address why the **relative reward formulation** (Eq. 2) — which uses ground truth as an anchor to prevent reward hacking on synthetic data — cannot be extended to real-world optimization. For synthetic data, the relative reward r(x₀) − r(ŷ₀) ensures the model improves relative to ground truth. For real-world data, Eq. 3 uses an absolute reward r(ŷ₀) with only DINO semantic alignment as the stabilizer. Section 3.2 states that "the lack of paired supervision makes the model particularly prone to over-optimizing the reward signal," but does not explain why a relative formulation is infeasible when no ground truth exists — one could, in principle, use a reference model's output as the anchor instead of ground truth (similar to how the semantic alignment already uses a reference model in Eq. 5). The paper treats the absolute-reward-plus-semantic-alignment approach as the only viable option, but this choice is not justified or ablation-tested. If the relative reward formulation were extended to real-world data using the reference model as anchor, would semantic alignment still be necessary? The answer to this question determines whether the paper's mitigation strategy is optimal or merely sufficient.

**[probe-evidence]** Could the authors clarify whether they attempted a relative-reward formulation for real-world optimization (using the reference model as anchor, analogous to Eq. 2), and if so, what empirical differences were observed compared to the absolute formulation of Eq. 3? If not, what is the theoretical or practical barrier to this alternative?

**My current calibration.** ICML Overall: 4 (Weak Accept) — the framework is well-motivated and the experiments are thorough, but the reward formulation asymmetry between synthetic and real-world optimization is an unresolved design choice that limits the paper's claims about reward-hacking mitigation.
